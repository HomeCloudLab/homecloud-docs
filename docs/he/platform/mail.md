# דואר (Mail)

HomeCloud Mail הוא שירות דואר ב-control plane מעל מנוע **Stalwart**.

## ארכיטקטורה

| שכבה | תפקיד |
|------|--------|
| Console `/console/mail` | רשימת תיבות + כרטיס מטא-דאטה של השירות |
| Console `/console/mail/{mailboxId}` | לקוח אימייל — 3 פאנלים בדסקטופ, מסך יחיד במובייל |
| `homecloud-api` `/accounts/{id}/mail/*` | JWT + metadata ב-Postgres |
| Stalwart (K3s) | SMTP/IMAP + תוכן ההודעה (מקור האמת) |

גופים, קבצים מצורפים ותיקיות **לא** נשמרים ב-Postgres — Stalwart הוא מקור האמת היחיד.

## ניווט בקונסול

אותו דפוס כמו SO / Queues: **רשימה → פרטי משאב**. פתיחת תיבה משאירה את מסגרת המייל גלויה בזמן הטעינה (בלי הבהוב skeleton של כל העמוד); ה-padding של ה-console נשאר יציב בכל אזור Mail.

1. **`/console/mail`** — טבלת תיבות, **צור תיבה**, **תבניות**, **אנשי קשר**, ופאנל סטטוס/DNS.
2. **`/console/mail/templates`** — ספריית תבניות HBS (ריק / ברכה / התראה / הודעה / מבצע / חשבונית).
3. **`/console/mail/templates/{id}`** — מעצב ממוקד קנבס (שכבות, מאפיינים, תצוגת מכשיר/כהה, בטל, לשונית מפתח).
4. **`/console/mail/contacts`** — ספר כתובות עם בחירה מרובה ומחיקה מרובה.
5. **`/console/mail/{mailboxId}`** — לקוח אימייל:
   - **דסקטופ / טאבלט** — פריסת 3 פאנלים: תיקיות, רשימה, קריאה/כתיבה; תצוגת תיבה כצ׳יפ בסגנון Gmail (אווטר + שם תצוגה + מייל מודגש); **הודעה חדשה** ו**מסך מלא** בסרגל מתחת לצ׳יפ (במסך מלא אותה זהות + יציאה)
   - **מובייל** — מסך אחד בכל רגע (רשימה → קריאה → כתיבה/הגדרות) עם אנימציית מעבר; תיקיות מ-☰ או מכותרת התיקייה (drawer); FAB לכתיבה; מקצה לקצה באמת (בלי padding של ה-shell); כתובות ארוכות נחתכות (`שם +N`); גלילה עד הסוף מעל ה-FAB וסרגל הסטטוס
   - **סרגל צד** — נכנס, יוצא, טיוטות, אשפה, ארכיון, חיפוש, הגדרות וכפתור כתיבה (בדסקטופ)
   - **רשימת הודעות** — אווטרים, נושא, תצוגה מקדימה, תאריכים יחסיים, נקודת "לא נקרא", אייקון צרופה; בחירה מרובה + העברה לאשפה (ומחיקה לצמיתות באשפה)
   - **תצוגת הודעה** — רינדור HTML מאובטח (DOMPurify) ב-iframe; מעטפות שיווק ברוחב קבוע (למשל טבלאות Brevo ב-600px) מתכווצות לרוחב הקורא; גלילה אופקית כשהתוכן עדיין חורג; fallback לטקסט; הורדת צרופות; סרגל פעולות

### אנשי קשר ו-bulk
- **אנשי קשר** — `/console/mail/contacts`; בחירה מרובה ומחיקה מרובה
- **כתיבה** — בחירת אנשי קשר ל-To (הודעה אחת לכמה נמענים)
- **API** — `GET/POST /contacts`, `POST /contacts/bulk-delete`, `POST /messages/bulk-trash`, `POST /messages/bulk-delete`

## תכונות לקוח אימייל

### כתיבה
- **עורך טקסט עשיר** (Tiptap) עם סרגל: מודגש, נטוי, קו תחתון, קו חוצה, גודל גופן, כותרות, רשימות, ציטוט, קישורים (חלון לכתובת + טקסט; משתמש בסימון אם יש), **תמונות/לוגו בתוך ההודעה** (סרגל, הדבקה או גרירה; נשלח כ-`cid:` כדי שיופיעו ב-Gmail), יישור, ניקוי עיצוב, בטל/חזור — מצב פעיל מסומן בסרגל; רשימות וציטוט מוצגים עם סגנון נראה
- **הוספת תבנית** — מקמפלת את התבנית ושומרת את **HTML המייל המלא** (טבלאות/סגנונות) בגוף הודעה לעריכת טקסט — בלי TipTap שמוחק את הפריסה.
- **כיוון טקסט** — בשליחה נעטף ה-HTML ב-`dir="rtl"` (או `ltr`) לפי שפת הממשק / תוכן עברי-ערבי כדי שגם ב-Gmail הכיוון יישאר כמו בעורך
- **צ׳יפים בסגנון Gmail** לנמענים To, CC, BCC — פרדה ברווח, פסיק, Enter; לחיצה על צ׳יפ לעריכת הכתובת; מחיקה עם Backspace או X; ניקוי תווי Bidi; שליחה נחסמת אם יש צ׳יפ לא תקין
- **כתובות נמענים** — תווי Unicode בלתי־נראים / Bidi (נפוץ בהעתקה מ-Gmail בעברית) מוסרים בצ׳יפים ובשליחה; כתובות לא תקינות נדחות לפני SMTP
- HTML body ו-plain text נשלחים ל-API
- **תמיכה בצרופות** — כפתור אטב פותח בוחר קבצים, גרירה ושחרור בכל אזור הכתיבה; קבצים מקודדים ב-base64 ונשלחים עם ההודעה
- חלקי bounce/DSN ריקים (0 בתים) מוסתרים מרשימת הצרופות
- אינדיקטור התקדמות העלאה בזמן שליחה עם צרופות
- קיצור Ctrl+Enter לשליחה

### השב / השב לכולם / העבר
- **השב** — ממלא To = השולח המקורי (כתובת נקייה), נושא = `Re: {subject}`, גוף מקורי כציטוט
- **השב לכולם** — ממלא את כל הנמענים בהסרת עצמי (כתובות מנוקות)
- **העבר** — נושא = `Fwd: {subject}`, כולל כותרת הודעה מועברת
- תשובות מגדירות `In-Reply-To` ו-`References` לשירשור שיחות תקין

### שירשור שיחות
- ה-Backend שולף כותרות `Message-ID`, `In-Reply-To` ו-`References` מ-Stalwart דרך IMAP
- הכותרות נשמרות ב-Postgres metadata לקיבוץ שרשורים

### מחיקה / אשפה / מחיקה לצמיתות
- מחיקה מעבירה הודעות לאשפה דרך IMAP (COPY + DELETE + EXPUNGE)
- שחזור מעביר הודעות מאשפה חזרה ל-INBOX
- מחיקה לצמיתות מסירה מ-Stalwart ומ-Postgres
- תיקיית אשפה מוצגת בסרגל הצד

### תיקיית יוצא
- הודעות שנשלחו נשלפות מתיקיית Sent של Stalwart דרך IMAP (לא רק metadata)
- גוף מלא וכותרות זמינים בצפייה בהודעות שנשלחו
- **תוצאת שליחה** — הצלחה פירושה ש-Stalwart **קיבל** את ההודעה למשלוח (לא הבטחה שהתיבה המרוחקת קיבלה)
- דחיית SMTP מיידית מוצגת כסטטוס `failed` עם סיבת השרת בממשק
- כשלים אסינכרוניים (למשל bounce מ-Gmail אחרי קבלה) מגיעים לתיבת הנכנס; כשה-bounce מפנה ל-`Message-ID` שנשלח, ההודעה היוצאת מסומנת `failed`

### Gmail / ספקים גדולים — הערות משלוח
- Gmail דוחה שליחה ישירה מ-IP ביתי / לא מורשה (`550 5.7.1 The IP you're using to send mail is not authorized…`). היציאה חייבת להיות משרת דואר עם **PTR + SPF + DKIM** (Stalwart על ה-IP הציבורי של המייל), לא מ-IP דינמי של ספק ביתי. אם רואים bounce כזה — בדקו `MAIL_PUBLIC_IP`, DNS, ושווידאתם ש-Stalwart הוא ה-MTA שיוצא החוצה.
- כתובות עם סימני RTL/Bidi (למשל `user@gmail.com` + U+202C/U+200F) גורמות ל-`DNS resolution error: Malformed label` — מטופל בניקוי כתובות בכתיבה ובשליחה.

### צרופות
- מטא-דאטה של צרופות שהתקבלו מחולצת מחלקי MIME ב-IMAP
- הורדת צרופות דרך בקשת blob מאומתת (לא קישור חשוף)
- רשימת צרופות מוצגת בתצוגת ההודעה עם שם, גודל וכפתור הורדה
- אייקון צרופה (אטב) ברשימת ההודעות
- כתיבה: בוחר קבצים + גרירה ושחרור, קידוד base64, נשלח כחלק מההודעה
- תמונות inline עם `cid:` מומרות לנתיבי הורדה לרינדור תקין

### חיפוש
- תיבת חיפוש בסרגל הצד מפעילה IMAP SEARCH דרך Stalwart
- חיפוש לפי נושא, שולח ונמען

### מעקב קריאה
- הודעות מסומנות כנקראו (`is_read`) בפתיחה דרך GET detail
- הודעות שלא נקראו מוצגות בטקסט מודגש ונקודה צבעונית
- תג מונה הודעות שלא נקראו בסרגל הצד

### רינדור אימייל HTML
- אימיילים נכנסים ב-HTML עוברים סניטציה עם DOMPurify ומוצגים ב-iframe מבודד
- **עיצוב נשמר** — פרופיל HTML משאיר תגיות פריסה (`center`, טבלאות, `font`, `style`/`align` של השולח); מוסרים רק תגיות פעילות/מסוכנות (`script`, טפסים, iframes וכו')
- fallback לטקסט פשוט עם `whitespace-pre-wrap`
- ריווח במעטפת הקורא (כרטיס); בתוך ה-iframe אין padding — הפריסה של השולח שולטת
- **מצבי כהה (Homelab / Midnight)** — iframe שקוף עם טקסט ברירת מחדל רך למיילים בלי עיצוב; רקעים לבנים / טקסט שחור כמעט מתרככים לקריאות בלי לשכתב לייאאוט
- בכתיבה יוצאת עדיין נעטף `dir` לפי הממשק כדי שעברית תישאר נכונה אצל לקוחות חיצוניים

### תיקיות
- נכנס, יוצא, טיוטות, אשפה, ארכיון — הכל מגובה בתיקיות IMAP של Stalwart

### הגדרות תיבה
- **שם תצוגה** — מוצג כשם השולח בהודעות יוצאות (כותרת From)
- **חתימה** — מוזרעת לעורך הכתיבה (אחרי `--`) כדי שאפשר לערוך או למחוק לפני שליחה; ה-API לא מוסיף אותה בשקט
- **העברה** — `forward_to` נשמר ומוחל על Stalwart דרך Sieve (`redirect :copy` שומר עותק מקומי)
- **בונה אוטומציה** — כרטיסי **מתי… אז…** מתקפלים עם סיכום חי; מספר תנאים (כל אלה / אחד מאלה) ומספר פעולות; גרירה לשינוי עדיפות; שכפול / הפעלה / ייצוא–ייבוא JSON; בדיקה מול הודעת דוגמה + תצוגת Sieve. מקומפל לאותו סקריפט Sieve כמו ההעברה. סידור אטומי: `POST …/mailboxes/{id}/rules/reorder` עם `{ "ids": [...] }`
- ההגדרות נגישות מלשונית "הגדרות" בסרגל הצד בכל תיבה
- הגדרות (וכתיבה) תופסות את כל חלק התוכן — רשימת ההודעות הריקה מוסתרת; הטופס נגלל עם שורת שמירה קבועה למטה

## תבניות אימייל (HBS)

תבניות ברמת חשבון לשימוש חוזר:

| משטח | נתיב |
|------|------|
| ספרייה | `/console/mail/templates` |
| עורך | `/console/mail/templates/{id}` |
| API | `GET/POST /accounts/{id}/mail/templates`, `PATCH/DELETE …/{template_id}`, `POST …/preview`, `POST …/{template_id}/render`, `POST …/{template_id}/duplicate` |
| שליחה | מצב תבנית בקומפוז, או `POST …/messages` עם `template_id` + `template_variables` אופציונליים |

מודל מסמך: בלוקים שטוחים ומקוננים כולל **`section`** (ילדים, עומק עד 3) למסגרות. תצוגה מקדימה בלי שמירה; לחיצה על הקנבס בוחרת בלוק.

### מנוע Responsive למייל (באחריות ה־Renderer)
כל קומפילציה מייצרת **HTML בטוח למייל (טבלאות)** שרספונסיבי כברירת מחדל (ADR-030). ההתנהגות מוגדרת ברמת רכיב ו**נאכפת ב־`mail_hbs`** — לא CSS ידני לכל תבנית.

| כלל | התנהגות |
|-----|---------|
| מעטפת | מקסימום ≈600px בדסקטופ; `width:100%` + ריפוד במובייל |
| נקודת שבירה | `@media only screen and (max-width:600px)` |
| תמונות | `width` + `max-width:100%;height:auto` |
| כפתורים | רוחב מלא במובייל (ביטול: `fullWidth: false`) |
| עמודות | ערמה במובייל כברירת מחדל |

תצוגת Desktop | Mobile | Tablet ב־Studio משתמשת ב**אותו** API קומפילציה; רוחב ה־viewport מפעיל את ה־media query.

### מעצב קנבס
- **Template Studio** (ספרייה): בונה מקצועי — שכבות, Insert, מאפיינים (כולל כוונת responsive), מכשיר/כהה, בטל, מפתח.
- **הכנסת תבנית ב־Compose:** מקמפלת את התבנית ו**מזריקה HTML לעורך TipTap הקיים** לעריכה חופשית (אותו ממשק כמו טיוטה רגילה). Studio לא נטען בתוך הכתיבה.
- שלדים ו־slots נשארים בצד Studio; ב־Compose אין טופס slots נעול.

### הכנסת תבנית לכתיבה
1. Insert template → קומפילציה דרך API → HTML נכנס ל־TipTap
2. נושא מתמלא מהתבנית; גוף ההודעה ניתן לעריכה חופשית
3. שמירת טיוטה / שליחה כמו כל הודעה רגילה

## הקשחת מסירה (Deliverability)

בקונסול: **סטטוס → מסירה** — בדיקות חיות ו־**Fix** ל-CloudNS כשמחובר.

| פריט | מומלץ | הערה |
|------|--------|------|
| SPF | `v=spf1 mx -all` | בלי `ip4:` — מתאים ל-IP דינמי בבית |
| DKIM | TXT מ-Stalwart | Fix DKIM מפרסם + מסנכרן zone |
| DMARC | `p=quarantine` + rua/ruf | צור תיבה לדוחות |
| TLS-RPT | `_smtp._tls` | דוחות כשלי TLS |
| MTA-STS | TXT + מדיניות HTTPS | `mta-sts.<domain>/.well-known/mta-sts.txt` |
| PTR | `mail.<domain>` | **רק אצל ספק ה-IP** (בסלקום ביתי לרוב אי אפשר) |
| BIMI | בהמשך | אחרי DMARC enforcement |
| ARC | Stalwart | בעיקר לשרשראות העברה |

סקריפט רענון DNS: `setup-mail-dns-homelab.sh`. פרטים מלאים בעמוד ה-EN.

## Config

ראה את עמוד ה-EN לפרטי env ודוגמאות API. פתחו בראוטר **25 / 465 / 587**; אל תחשפו את פורט הניהול הציבורי.

!!! note "Stalwart 0.16 — listeners"
    ב-Stalwart 0.16 הגדרות listeners נשמרות ב-data store ולא ב-`config.json`.
    IMAP (143) ו-submission (587) **לא** נוצרים בברירת מחדל — יש ליצור אותם
    דרך JMAP (`x:NetworkListener/set`) או WebUI ולהפעיל מחדש את ה-pod.
    ה-Helm chart חושף `hostPort: 143` ו-`hostPort: 993` כדי שה-API על ה-host יגיע ל-IMAP.
